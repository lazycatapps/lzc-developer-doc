# LightOS 使用场景

LightOS 用于需要长期维护完整 Linux 运行环境的场景。它不是 LPK 的替代品：LPK 适合把应用封装成可分发、可复现的一键安装包；LightOS 适合由用户或管理类应用维护更接近系统环境的实例。

## 什么时候使用 LightOS {#when-to-use-lightos}

如果你的需求符合以下情况，优先使用 LightOS：

1. 需要通过包管理器安装软件，并希望重启后继续保留。
2. 需要长期保存系统级配置、开发工具链、脚本、服务进程或调试环境。
3. 需要一个可进入 shell 的完整 Linux 环境，而不是只运行一个封装好的 lzcapp。
4. 需要把 SSH 中临时验证过的命令、配置或工具沉淀为可长期使用的环境。

不要把 LightOS 当作普通应用分发方式。面向普通用户交付独立应用时，仍应优先制作 LPK。

## 如何使用 LightOS {#how-to-use-lightos}

普通用户不需要手工配置系统组件。打开微服应用商店，搜索并安装 LightOS 入口应用，然后在 LightOS 页面中创建和管理实例即可。

LightOS 实例内的系统、软件和配置会按实例持久保存，适合替代直接 SSH 到 `lzcos` 后安装软件的用法。`lzcos` 的 SSH 环境是 read-only 系统，重启后通过 SSH 对系统做的变动会丢失。

LightOS 拥有较高权限，只应开放给可信用户或可信应用。
