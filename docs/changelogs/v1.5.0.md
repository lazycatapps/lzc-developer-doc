v1.5.0 未发布

# 兼容性调整

- `/lzcapp/document` 和 `/lzcapp/run/mnt/home` 已标记为废弃（deprecated），后续仅保持兼容，文稿访问路径为 `/lzcapp/documents`。
- 为兼容历史应用，`/lzcapp/document` 与 `/lzcapp/run/mnt/home` 的挂载权限处理保持旧行为，不使用 idmapped 挂载。

# 兼容性破坏说明

- `/lzcapp/var` 的挂载权限模型已调整为按 service 运行 UID/GID 进行 idmapped 挂载，因此同一应用新版本如果修改了 `services.<name>.user` 或镜像 `USER`，同一份历史数据在容器内看到的 owner 语义会变化；依赖 setup_script 对 `/lzcapp/var` 执行 `chown` 作为长期权限控制的应用，需要改为通过 `services.<name>.user` 声明运行用户。
